HBox Basics

Webbook has another concept for horizontal box layout. This is more complexity for little use unless you want broken positioning e.g. left/right placement, or dropdown menus.

These classes can not be used on lists. Lists come with awkward preset spacings. Lists add too much extra HTML structure, and their preset spacing is too much to kill.

Three base boxes are provided, with alignment in different directions. Contents can be near-anything, classed as 'hbox-item'. Box-items take on the size of their contents. All are responsive. They break on paper-web width. Left,

Drop Menus

A drop menu can also be built. This needs the HTML form defined below, and the included snippet of JS.

For narrow/vertical display the usual size for menu buttons is button-sl. For narrow/vertical consistency, all buttons should be the same size. But in the horizontal direction, the horizontal buttons probably need more vertical size. hbox-button has the same wide-display width as button-sl and button-nl, so can be used for root menu items. Drop menu items will use button-sl or button-nl. If you want to construct your own breaking button for another horizontal width, keep it the same width as the drop menu sizes, or text alignment will break,
